Skill surface — invocation taxonomy and roster

A skill is a small procedure document the agent reads and follows. Below is everything the seven plugins bring, grouped by plugin.

33 SKILL.md files across the 7 plugins, three invocation kinds: user slash commands, model-invoked, and orchestrator-dispatched. The same files project onto Codex via the adapter.

The skill surface is the host-portable half of the runtime contract: plain-markdown procedure files, discovered natively by Claude Code and projected losslessly onto Codex.

estregenesis 5 procedures + 4 aliases

How to use it — per environment

Invocation surfaces by host environment

You can use EstreGenesis in three places: the Claude Code terminal (CLI), the Claude Code panel inside VS Code, and — through a small included adapter — OpenAI Codex.

Three host environments: Claude Code CLI + VS Code (identical plugin behaviour), and OpenAI Codex via the bundled adapter (skills + MCP project losslessly; hooks map to a documented manual discipline).

Three host surfaces: the two Claude Code surfaces share an identical plugin runtime; OpenAI Codex is served by a projection adapter — skills and MCP port losslessly, the hook lifecycle degrades to a documented paired-skill discipline.

Use the same EstreGenesis skills from OpenAI Codex — a small adapter that ships inside the repo sets everything up.

Projection adapter: gen-codex-adapter.cjs materializes the canonical SKILL.md set on demand and regenerates only the derived surfaces (config.toml stanzas + inventory). Hooks map to a documented paired-skill discipline via codex/AGENTS.md.

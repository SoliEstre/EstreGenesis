How you'll use this

How to use it — per environment

Invocation surfaces by host environment

Claude Code runs in two environments: the terminal (CLI) and VS Code. All EstreGenesis plugins work the same way in both. The entry point is what changes.

Claude Code ships in two host environments — terminal CLI and VS Code extension. Plugin behaviour is identical; only entry-point + chat-surface affordances differ.

Claude Code exposes two host surfaces sharing an identical plugin runtime; they differ exclusively in session bootstrap and chat-affordance rendering.

Claude Code CLI

Run Claude Code directly from the terminal.

Direct-from-terminal session. Best fit for headless workflows, CI integration, tmux-based multi-agent setups.

Terminal-resident host with stdin/stdout REPL and full shell integration. Preferred for scripted invocation and shell-managed orchestration.

  1. Start Claude Code in the terminal: claude or claude code
  2. Register the marketplace: /plugin marketplace add SoliEstre/EstreGenesis
  3. Install your chosen plugin: /plugin install constellation@estregenesis-plugins (or any other module name)
  4. Use slash commands, skills, and hooks in your session.
  1. Launch a CLI session: claude (or claude code) from your project root.
  2. Register the marketplace (one-time, idempotent): /plugin marketplace add SoliEstre/EstreGenesis
  3. Install per-plugin: /plugin install <name>@estregenesis-plugins — any subset of the six modules.
  4. Slash commands, skills, and hooks are now active — plugin hooks fire on lifecycle events (SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, Stop, PostCompact).
  1. Instantiate a CLI host process: claude or claude code — the process inherits the invoking shell's cwd as the workspace root.
  2. Persist the marketplace source descriptor (idempotent across re-issuance): /plugin marketplace add SoliEstre/EstreGenesis
  3. Admit individual plugins via fully-qualified namespace reference: /plugin install <name>@estregenesis-plugins — admission graph is flat (zero cross-plugin dependency edges).
  4. The 7-event lifecycle drives slash dispatch, skill registry, and hook execution; semantics are host-uniform with the VS Code surface.

Claude Code VS Code Extension

Use Claude Code as a panel inside VS Code.

VS Code extension surface — sidebar chat panel + integrated editor diff view. Best fit for IDE-resident workflows where the agent operates alongside open editor buffers.

VS Code extension host backed by the LSP-adjacent extension API. Plugin runtime semantics are bit-identical to the CLI host; only chat-surface rendering and session bootstrap differ.

  3. Register the marketplace: type /plugin marketplace add SoliEstre/EstreGenesis in the chat window
  4. Install plugins: use /plugin install <name>@estregenesis-plugins, then access modules from the sidebar.

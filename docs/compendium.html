What Compendium is

What Compendium is

Overview

Gathers cross-module terms, gives each a plain + an expert label tied to one meaning, and points back to the source rather than copying it — so it never drifts.

A concept-anchored dual-register glossary + cross-module wiki. One concept = one definition + register-tagged terms; internal entries are pointer-only into the owning spec, general entries (no owner) carry the full def.

A concept-anchored dual-register substrate; register is a property of the term/gloss, never of the canonical definition. Internal entries are pointer-only (machine-checked owner_spec); general entries carry the full def. One-directional authority makes the content layer clean-evictable.

The core idea

Central thesis

Central thesis

Point, don't copy — a glossary that points to the source can't go stale.

Vocabulary outlives runtime — so the durable investment is the concept-anchored model + the content-boundary charter, and the runtime/spec is the first prunable target.

North-star axis-2 (concept survival > code survival): the durable value is the vocabulary, so the runtime/spec is the first prunable target — passing its own eviction test by construction.

Taking terms out, not just adding them

Eviction is first-class

Eviction & self-eviction

Never a silent delete — a renamed term is marked 'replaced' with a pointer, so old links work. And it plans its own retirement: entries migrate cleanly, the bespoke runtime/spec is pruned first.

First-class deprecation (active/superseded/archived + superseded_by, isomorphic with the Greatpractice retire-axis). Explicit self-eviction: content migrates, the bespoke runtime/spec is pruned first.

First-class deprecation (skos:replacedBy), never deletion. A stated prune order — content → lints → MCP → spec — so the module that sells eviction models its own.

When it's worth adopting

Adoption thresholds

Adoption thresholds

Not for every project — earns its keep only with several modules, ~25+ cross-referenced terms, a live dashboard, and real drift evidence. For a single-doc or solo one-off, a plain index is the answer.

Adopt only when all hold: ≥2 overlapping specs, ~25+ active cross-referenced terms, a live dashboard, recurring drift. EG qualifies on all four. Below the bar, inline parentheses + a flat index are correct.

A conjunctive gate (four conditions), isomorphic with the Greatpractice/Superscalar thresholds; the negative-ROI case is stated explicitly so the module is not mis-adopted as a universal default.
